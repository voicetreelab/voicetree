Folder handle — interactive mockup

This page wires the real FolderHandleService from webapp/src/shell/UI/cytoscape-graph-ui/services/folder-handle/ into a stand-alone cytoscape canvas. The only stubs are window.electronAPI (so the chevron's collapse/expand IPC has somewhere to land) and the sample graph.

  • Chevron click → real toggleFolderCollapse path → stub IPC → cy mutation.
  • Folder body is ungrabify()'d: pan + right-click pass through to the canvas.
  • Hover the folder body → no grab cursor (matches the shipped mouseover early-return).
  • Collapsed folder = 40×40 pill, grabbable, no DOM chip (the cy node IS the chip).
  • Click a collapsed pill → expand. The stub rebuilds cy (see note); production routes via daemon IPC.

Heads-up: in-place expand stresses a compound-bounds re-entrancy in the shipped chip listener (positionChip reads bbox synchronously inside its own bounds event handler). Production hides this via daemon-async IPC + applyGraphDeltaToUI's remove/re-add cycle; the stub here rebuilds the cy instance instead — same end state.
