REV. B · COMPOSITE

Each sealed folder exposes a mode selector in its control bar — three keyed segments (drill, surface, lens) — that decides what Enter does on the focused row. Modifier keys allow temporary overrides without changing the persistent mode. Each row publishes the action it will take as a colored chip in the right slot, so the consequence of pressing Enter is visible before the keystroke. A per-folder default can be configured (e.g. tasks/ defaults to SURFACE because users always want a task subfolder to live in the graph; specs/ defaults to DRILL because users browse without committing).

per-folder defaults

Each folder remembers its own mode. A folder of tasks where every entry deserves graph-level prominence defaults to SURFACE; a folder of specs the user browses without commitment defaults to DRILL; an archive folder with hundreds of files defaults to LENS because no other mode scales there.

visible consequence

The per-row colored chip is non-negotiable: it shows before the keystroke what the consequence will be. This is the single most important UX guarantee — the user is never surprised by a graph mutation triggered by Enter.

cost & risk

Hybrid is the heaviest implementation: three runtime modes, modifier matrix, per-folder persistence, distinct visual languages. Recommend shipping DRILL first (cheapest, lowest blast-radius), validate, then add LENS if BF-112 confirms scale need, finally add SURFACE only if user research shows users want graph promotion as a primary action.
