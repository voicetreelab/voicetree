D4
arrow-key scope — when do arrows mean "next sibling" vs "pan canvas"?

The arrow-key keyboard is precious real estate. Two patterns are honest; the third is too clever. Recommendation: arrows are focus-contextual — they do whatever the focused element wants. The filetree drawer claims them when focused; the canvas claims them otherwise.

Esc always escapes one level (drawer → folder → parent folder → canvas). ⌘P / Ctrl+P opens a fuzzy filename search over the entire filesystem (loaded or not) — this is the answer to "I just want to find a file fast, anywhere."

  1. Outer container: graph is outer. No sidebar. (D1 = B)
  2. Filetree home: bottom drawer inside each expanded folder body. Max-height + scroll + filter at >30 entries. (D2 = A)
  3. Loaded vs unloaded: ▣ (gold, filled) vs □ (cyan, outlined). Glyph + color, never color alone. (D3 = A)
  4. Arrow keys: focus-contextual. Drawer claims them when focused; otherwise canvas pan. ⌘P for global fuzzy filename search. (D4 = focus-scoped)
  5. Load trigger: explicit only — ↵ on focused row, or single click. Never hover, never scroll. (D5 = A)
