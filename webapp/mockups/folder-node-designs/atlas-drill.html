A sealed folder is no longer a single black-box icon — it is a compact, keyboard-navigable file-tree. ↑↓ moves the focus row, → / Enter drills into the focused subfolder (the list re-renders one level deeper, breadcrumb pushed), ← / Esc pops back up. The surrounding graph is never touched. The folder still publishes its edge interface (the three-port fan on the right border) regardless of how deep the user has drilled.

contract

Drilling never alters the graph. The sealed folder remains sealed; the breadcrumb is local state. The external node positions, the edge-fan, and the cross-folder synthetic edges are stable throughout drill operations. This is the cheapest credible answer to BF-112 — users navigate huge subtrees without paying graph-layout cost.

escape hatch

Shift+Enter on any focused row breaks the contract by explicit user action — it expands the parent folder into the graph (or, if the focused row is a subfolder, expands that instead). This is the only way to mutate the graph from inside drill mode, and it is keyboard-explicit.

visual debt

The sealed-folder height now scales with the visible row count (8 rows visible by default, scrollable beyond). On a dense graph this competes for canvas space. The mitigation is the breadcrumb-only-when-drilled rule: at depth 0, the breadcrumb bar is suppressed.
