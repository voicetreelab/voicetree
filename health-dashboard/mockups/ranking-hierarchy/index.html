Notes for the picker

All four pull from the live reports/latest.json. Aggregation rule is identical: parent · util = max(child · util) — the worst-first principle. Subgroups exist only where ≥ 2 metrics share a semantic prefix (globals-, dsm-, hypergraph-, etc.); single-metric subgroups attach directly to their category to keep depth honest. Categories themselves are sorted worst-first too — Purity (∞×, gate fail) tops the list.
